World Country API Documentation

Welcome to the developer documentation for the World Country Explorer API. Use these endpoints to access, search, and modify country data. All data is returned in application/json format.

Endpoint Summary

Endpoint Methods Description Success Code
/getAll GET, HEAD Retrieve all countries in the dataset. 200 OK
/getById GET, HEAD Retrieve a single country by name. 200 OK
/search GET, HEAD Filter countries by name or region. 200 OK
/meta GET, HEAD Get count of total items in dataset. 200 OK
/addItem POST Add a new country record. 201 Created
/editItem POST Update an existing country record. 204 No Content

GET /getAll

Returns the full list of countries currently stored in the server's memory.

  • Supported Methods: GET, HEAD
  • Query Params: None
  • Returns: An object containing an items array.
  • Example Usage: GET /getAll
Example Response: 
{
  "items": [
    { "id": "1", "name": "Canada", "capital": "Ottawa", "region": "Americas" },
    { "id": "2", "name": "France", "capital": "Paris", "region": "Europe" }
  ]
}

GET /getById

Fetches data for a specific country based on its name.

  • Supported Methods: GET, HEAD
  • Query Params:
    • name (String, Required): The name of the country.
  • Returns: A single country object.
  • Example Usage: GET /getById?name=Japan
Example Response: 
{
  "id": "16234",
  "name": "Japan",
  "capital": "Tokyo",
  "region": "Asia"
}

GET /search

Search and filter countries by name fragments or geographical region.

  • Supported Methods: GET, HEAD
  • Query Params:
    • name (String, Optional): Case-insensitive partial name match.
    • region (String, Optional): Exact case-insensitive region match.
  • Returns: An array of matching objects inside an items key.
  • Example Usage: GET /search?region=Europe&name=ice
Example Response: 
{
  "items": [
    { "id": "55", "name": "Iceland", "capital": "Reykjavik", "region": "Europe" }
  ]
}

POST /addItem

Creates a new country entry. Requires JSON or URL-encoded body data.

  • Supported Methods: POST
  • Body Parameters:
    • name (String, Required)
    • capital (String, Required)
    • region (String, Required)
  • Returns: The newly created object.
  • Example Usage: POST /addItem
Example Body (JSON): 
{
  "name": "Atlantis",
  "capital": "Poseidonia",
  "region": "Ocean"
}
Example Response (201 Created): 
{
  "id": "1710438291000",
  "name": "Atlantis",
  "capital": "Poseidonia",
  "region": "Ocean"
}

POST /editItem

Updates an existing entry based on the provided name.

  • Supported Methods: POST
  • Body Parameters:
    • name (String, Required): The current name of the country to find.
    • newName (String, Optional): The new name to assign.
    • capital (String, Optional): The new capital name.
    • region (String, Optional): The new region name.
  • Returns: No content (Status 204).
  • Example Usage: POST /editItem with body {"name": "Atlantis", "capital": "New Atlantis"}

Error Responses

Common error codes returned by the API:

  • 400 Bad Request: Missing required parameters.
  • 404 Not Found: The requested endpoint or country name does not exist.
Example Error Body: 
{
  "message": "Name parameter is required",
  "id": "missingName"
}
